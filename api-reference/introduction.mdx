The Unified Platform API uses standard HTTP verbs and status codes, accepts and returns JSON, and follows consistent patterns across all resources.

## Base URL

```
https://api.platform.io/v1/
```

## Authentication

| Method | Header | Use case |
|--------|--------|----------|
| JWT Bearer | `Authorization: Bearer <jwt>` | User sessions (RS256, 15 min access / 7 day refresh) |
| API Key | `Authorization: Bearer sk_live_...` | Server-to-server (`sk_test_` for sandbox) |
| Widget Session | `X-Widget-Session: ws_...` | Embeddable booking widget (30 min TTL) |

JWT claims include `tid` (tenant), `lid` (location), and `roles[]`.

## Pagination

List endpoints use ID-based cursor pagination:

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Items per page (1–100, default 10) |
| `starting_after` | string | Object ID — page after this object |
| `ending_before` | string | Object ID — page before this object |

Response envelope:

```json
{
  "object": "list",
  "url": "/v1/customers",
  "has_more": true,
  "data": [...]
}
```

## Expanding objects

Related objects return as IDs by default. Expand inline with `?expand[]=field`:

```
GET /v1/orders/ord_123?expand[]=customer&expand[]=bookings
```

Max depth: 4 levels.

## Errors

```json
{
  "error": {
    "type": "validation_error",
    "code": "BOOKING_SLOT_UNAVAILABLE",
    "message": "The requested time slot is no longer available.",
    "param": "slot_id",
    "request_id": "req_abc123",
    "doc_url": "https://docs.platform.io/errors/BOOKING_SLOT_UNAVAILABLE"
  }
}
```

Error types: `api_error`, `validation_error`, `business_error`, `idempotency_error`.

## Idempotency

POST requests accept an `Idempotency-Key` header (UUID). Keys are cached for 24 hours. Reusing a key with different parameters returns 409 Conflict.

## Versioning

Primary versioning uses the URL path (`/v1/`). Per-request override via `X-API-Version` header.

## Request IDs

Every response includes a `Request-Id` header. Include this when contacting support.

## Metadata

Most mutable resources accept `metadata` — up to 50 key-value string pairs for storing structured data.
